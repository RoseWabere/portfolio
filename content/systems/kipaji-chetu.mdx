## Problem Context

In many classrooms, especially in resource-constrained environments, teachers manage large groups of students with diverse learning abilities. Providing personalized attention, adaptive learning content, and timely feedback becomes nearly impossible.

Most existing learning platforms focus on static content delivery and repetitive drills, which leads to:
- Low student engagement over time
- Limited personalization
- Lack of actionable insights for teachers
- Poor accessibility for learners with different needs

**The core challenge:**  
How do you build a system that adapts to each student’s learning pace, provides real-time feedback, and generates meaningful insights — all while remaining lightweight and accessible?

## System Architecture


Student → FastAPI Backend → AI Engine (Groq) → PostgreSQL → Analytics → Response
↓
TTS / STT Layer


**Core Components:**
1. **Learning Engine:** Adaptive quiz generation and answer evaluation
2. **AI Layer:** Generates questions, feedback, and simplified explanations
3. **Database Layer:** Stores student performance, attempts, and analytics
4. **Analytics Engine:** Tracks progress and identifies learning patterns
5. **Accessibility Layer:** Text-to-speech and speech-to-text support
6. **Frontend:** Lightweight HTML/JS interface for students and teachers

Tech Decisions & Tradeoffs
Decision 1: Groq vs OpenAI

Chosen: Groq

Faster inference
Free tier availability
Zero data retention

Tradeoff:

Slightly less ecosystem maturity than OpenAI
Decision 2: FastAPI vs Flask

Chosen: FastAPI

Async support for AI calls
Built-in OpenAPI docs
Better performance under concurrency
Decision 3: PostgreSQL vs SQLite

Chosen: PostgreSQL

Supports scaling
Better for analytics queries
Handles concurrent users

What I Learned
Adaptive systems outperform static content. Personalization significantly improves engagement and learning efficiency.
AI must be controlled. Without structured prompts and validation, outputs become unreliable.
Backend design matters in AI systems. Data flow and storage are as critical as model quality.
Accessibility is not optional. Simple features like TTS dramatically improve usability.
Real-time feedback changes behavior. Immediate explanations reinforce learning better than delayed results.
