## Overview

The Kenya Financial Intelligence System is a backend platform designed to give
Kenyan retail investors — from M-Pesa mobile-first users to SACCO members and
young Nairobi professionals — trusted, actionable financial guidance in one
place. It combines a Retrieval-Augmented Generation (RAG) Q&A engine, a
rule-based investment decision engine, an automated data pipeline, and a custom
WhatsApp bot, all served through a FastAPI REST API.

The system answers questions like:
- *"Best MMF for a beginner with KES 5,000?"*
- *"Is Stima SACCO registered with SASRA?"*
- *"I earn KES 45k/month. How should I invest for a house in 7 years?"*

## RAG Pipeline — How It Works

Each chat message goes through three retrieval modes before hitting Claude:

```
User message
    │
    ▼
1. EMBED query (sentence-transformers, 384 dims)
    │
    ▼
2. pgvector cosine similarity search
   → Top 5 knowledge_chunks (KB articles, guides, regulation)
   → Top 3 news_articles (recent finance news)
    │
    ▼
3. Structured DB injection (if query contains: broker/fund/sacco/bond/mmf/reit)
   → Fetch verified providers from DB
   → Append as plain-text context block
    │
    ▼
4. Build context string + fetch last 10 chat turns (session history)
    │
    ▼
5. POST to Claude API (claude-sonnet-4-20250514)
   System prompt: Kenya-first, cite sources, flag unregulated products,
                  plain English + optional Kiswahili, disclaimer on every reply
    │
    ▼
6. Persist ChatSession + ChatMessages + QueryLog
    │
    ▼
Return {answer, sources, confidence, session_id}
```

**Confidence score** = average cosine similarity of retrieved chunks. Low confidence answers include a disclaimer.

## Decision Engine — Allocation Rules

The engine maps a `UserProfile` to an explicit allocation plan. Every rule is traceable:

| Signal | Rule |
|--------|------|
| `income < 30k` | 50% MMF emergency buffer first |
| `has_debt = true` | Add "Debt clearance" note; increase MMF |
| `risk = low` OR `horizon < 3yr` | 35% bonds/T-bills; no equity |
| `goals contains 'house'/'land'` | Add SACCO (loan leverage for property) |
| `horizon >= 5yr` AND `risk = medium/high` | Add NSE equities / unit trusts |
| `tier = high` AND `risk = medium/high` | Add REIT exposure |
| `age < 30` AND `horizon >= 5yr` | Higher equity weight |
| All cases | Final allocation normalised to exactly 100% |

No ML — every decision can be read and explained to a regulator.

## Changing the Anthropic API Key

The API key is read **once at startup** from the environment. To rotate it:

### In Docker Compose (local / self-hosted)

1. Open `.env`
2. Replace the line:
   ```
   ANTHROPIC_API_KEY=sk-ant-old-key
   ```
   with:
   ```
   ANTHROPIC_API_KEY=sk-ant-new-key
   ```
3. Restart only the backend and worker:
   ```bash
   docker compose restart backend worker
   ```
   The database and WhatsApp bot do not need a restart.

### In Render (production backend)

1. Go to your Render service dashboard → **Environment**
2. Find `ANTHROPIC_API_KEY` → click Edit → paste new key → Save
3. Render will trigger an automatic redeploy (takes ~2 minutes)
4. No downtime for the database (Aiven) or WhatsApp bot

### In Aiven / any managed platform

Same pattern — update the environment variable in the platform's dashboard or secrets manager. The key is only referenced in:
```
backend/app/core/config.py  →  ANTHROPIC_API_KEY: str
backend/app/services/rag_service.py  →  anthropic.AsyncAnthropic(api_key=settings.ANTHROPIC_API_KEY)
```

Those two files are the only places the key is consumed. No hardcoding anywhere.

### To switch LLM provider entirely

In `backend/app/services/rag_service.py`, the Claude call is isolated in the `answer_query()` function:

```python
# Current (Anthropic Claude)
_claude = anthropic.AsyncAnthropic(api_key=settings.ANTHROPIC_API_KEY)

response = await _claude.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1000,
    system=SYSTEM_PROMPT,
    messages=messages,
)
answer = response.content[0].text
```

To swap to OpenAI, replace those lines with:
```python
from openai import AsyncOpenAI
_client = AsyncOpenAI(api_key=settings.OPENAI_API_KEY)

response = await _client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "system", "content": SYSTEM_PROMPT}] + messages,
)
answer = response.choices[0].message.content
```

To swap to Groq (LLaMA):
```python
from groq import AsyncGroq
_client = AsyncGroq(api_key=settings.GROQ_API_KEY)

response = await _client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=[{"role": "system", "content": SYSTEM_PROMPT}] + messages,
)
answer = response.choices[0].message.content
```

Add the corresponding key to `.env` and `config.py`. Everything else stays the same.

**Key pages / components:**

1. **`/` Home** — Hero with chat input front and centre. "Ask anything about investing in Kenya."
   Subtitle: "CMA-verified data. No bias. No ads." Trust badges (CMA, CBK, SASRA logos).

2. **`/chat`** — Full chat UI. Sources shown as citation chips below each answer.
   Confidence bar. Session history in sidebar. Mobile-first (WhatsApp muscle memory).

3. **`/plan`** — Multi-step profile form (income, risk, goals, horizon).
   Step 4: animated allocation donut chart. Actionable steps with provider cards.

4. **`/directory`** — Searchable, filterable table of providers and investments.
   CMA-verified badge. Risk pill (green/amber/red). Min investment chip.

5. **`/market`** — NSE ticker cards (sparkline + daily change).
   Macro strip at top: CBR | Inflation | 91d T-bill yield — updated daily.

6. **`/learn`** — Glossary cards with Kiswahili terms. Searchable.
   Article cards linking to expanded KB content.

```typescript
// Every API call uses a shared fetch wrapper
export async function apiFetch<T>(
  path: string,
  options?: RequestInit,
  token?: string
): Promise<T> {
  const res = await fetch(`${API_BASE}${path}`, {
    ...options,
    headers: {
      "Content-Type": "application/json",
      ...(token ? { Authorization: `Bearer ${token}` } : {}),
      ...options?.headers,
    },
  });
  if (!res.ok) throw new Error(await res.text());
  return res.json();
}
```

## Running Locally

```bash
# 1. Clone and configure
git clone https://github.com/Rozieroz/kenyan-financial-intelligence
cd kenyan-financial-intelligence
cp .env.example .env
# Edit .env: add ANTHROPIC_API_KEY and NEWS_API_KEY

# 2. Start all services
docker compose up --build

# 3. Run migrations + seed KB (first time only)
docker compose exec backend alembic upgrade head
docker compose exec backend python scripts/seed_kb.py

# 4. Connect WhatsApp
docker compose logs -f whatsapp-bot
# Scan the QR code printed in the logs with your WhatsApp
# Session is saved to Docker volume — scan once only

# 5. Open API docs
open http://localhost:8000/docs

# 6. Test the recommendation endpoint
curl -X POST http://localhost:8000/api/recommendation \
  -H "Content-Type: application/json" \
  -d '{"income_kes_monthly":50000,"risk":"medium","goals":["house"],"horizon_years":7,"age":30,"has_debt":false}'
```

## Deployment

### Backend → Render

```bash
# Push backend/ as a Docker service on Render
# Set all .env variables in Render → Environment tab
# DATABASE_URL → your Aiven PostgreSQL connection string
# Health check path: /health
```

### Database → Aiven PostgreSQL

```bash
# Enable pgvector extension (Aiven supports it natively on PG15+)
# Use the Aiven connection string in DATABASE_URL and DATABASE_URL_SYNC
# Aiven encrypts at rest and in transit by default
```

### Frontend → Vercel

```bash
# Next.js frontend deploys to Vercel automatically on push
# Set NEXT_PUBLIC_API_URL=https://your-render-service.onrender.com
```

### WhatsApp Bot → VPS (DigitalOcean / Hetzner)

```bash
# whatsapp-web.js requires a persistent process (not serverless)
# Cheapest option: $4/month Hetzner CX11 VPS
# docker compose up whatsapp-bot -d
# docker compose logs -f whatsapp-bot  → scan QR once
# Session persists in Docker volume
```
